# talea-mappings

Community MIDI controller mappings for [Talea](https://talea.eurico.ws),
the open-source DJ app.

One catalog, two frontends: [`index.json`](index.json) feeds both the
[mappings page on the website](https://talea.eurico.ws/mappings.html) and the
in-app browser (Settings → MIDI → Get mappings). It is regenerated by CI on
every merge — **don't edit it by hand**.

## Install a mapping

- **In the app** (easiest): Settings → MIDI → Get mappings, pick your
  controller.
- **Manually**: download the mapping's `.json` from `mappings/`, then in Talea
  open Settings → MIDI → Import mapping… and pick the file.

## Contribute a mapping

1. Map your controller in Talea — MIDI learn does most of the work — and
   export the mapping.
2. Add two files in a PR:

   ```
   mappings/<manufacturer>/<controller>.json       # the exported mapping
   mappings/<manufacturer>/<controller>.meta.json  # catalog metadata
   ```

3. Once merged, CI regenerates `index.json` and the mapping appears on the
   website and inside every copy of the app — automatically.

CI validates both files on the PR (`node scripts/build-index.mjs` runs the
same checks locally).

### `.meta.json` schema

```json
{
  "id": "pioneer-ddj-400",
  "name": "Pioneer DDJ-400",
  "manufacturer": "Pioneer DJ",
  "controller": "DDJ-400",
  "author": "Your Name",
  "description": "One line on what the mapping covers.",
  "version": "1.0.0"
}
```

`id` is unique and kebab-case; `version` is semver. `author` and
`description` are optional but appreciated.

### Mapping file format

The mapping `.json` is exactly what Talea exports/imports: a `device` name,
`bindings` (a MIDI `[status, data1]` pair → a control or action target), and
optional `outputs` (LED feedback). The format is defined by the app in
[`crates/mixeee-controllers/src/mapping.rs`](https://github.com/eeeurico/talea/blob/main/crates/mixeee-controllers/src/mapping.rs);
[`mappings/pioneer/ddj-400.json`](mappings/pioneer/ddj-400.json) is a complete
example.
